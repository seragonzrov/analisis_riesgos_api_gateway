# Prompt 08: Matriz de Controles NIST/ISO 27001

**Fecha:** 24/05/2026
**Herramienta:** Claude Sonnet 4.6 (Anthropic)
**Propósito:** Mapear los controles de seguridad del sistema al NIST SP 800-53 e ISO 27001

---

## Prompt Utilizado

```
Eres un consultor de seguridad especializado en frameworks de compliance.
Contexto del sistema en 00-memory-bank.md.

Tenés los 15 controles de seguridad diseñados en docs/07-plan-mitigacion.md
(C01-C15) y las 27 amenazas STRIDE en docs/04-analisis-stride.md.

Generá una tabla de controles alineada a NIST SP 800-53 e ISO 27001:2022.

Para cada control del sistema, identificá el control NIST SP 800-53
correspondiente y su equivalente ISO 27001, con una descripción de cómo
se implementa concretamente en este sistema.

Cubrí las siguientes familias de controles NIST, una fila por control:

Familia AC (Access Control):
- AC-1: Access Control Policy
- AC-2: Account Management
- AC-3: Access Enforcement
- AC-6: Least Privilege

Familia AU (Audit and Accountability):
- AU-2: Audit Events
- AU-6: Audit Review and Analysis
- AU-9: Protection of Audit Information

Familia CA (Assessment, Authorization and Monitoring):
- CA-2: Security Assessments
- CA-7: Continuous Monitoring

Familia CM (Configuration Management):
- CM-2: Baseline Configuration
- CM-3: Configuration Change Control
- CM-7: Least Functionality

Familia CP (Contingency Planning):
- CP-9: System Backup

Familia IA (Identification and Authentication):
- IA-2: Identification and Authentication (Organizational Users)
- IA-5: Authenticator Management

Familia IR (Incident Response):
- IR-4: Incident Handling
- IR-6: Incident Reporting

Familia RA (Risk Assessment):
- RA-5: Vulnerability Monitoring and Scanning

Familia SC (System and Communications Protection):
- SC-7: Boundary Protection
- SC-8: Transmission Confidentiality and Integrity
- SC-12: Cryptographic Key Establishment and Management
- SC-13: Cryptographic Protection
- SC-28: Protection of Information at Rest

Familia SI (System and Information Integrity):
- SI-2: Flaw Remediation
- SI-3: Malicious Code Protection
- SI-4: System Monitoring

Para cada fila de la tabla, incluí:
- ID del control NIST (ej: AC-1)
- Nombre del control
- Descripción de cómo se implementa en este sistema (1-2 líneas)
- Referencias: NIST y ISO 27001 correspondiente

Formato Markdown con tabla y navegación al final.
```

---

## Response Summary

La IA generó una tabla de **26 controles** distribuidos en 10 familias NIST:
- **AC (Access Control):** AC-1 (RBAC K8s + Istio AuthZ), AC-2 (SSO + provisioning
  automatizado), AC-3 (Admission controllers + OPA), AC-6 (service accounts únicos
  por microservicio)
- **AU (Audit):** AU-2 (K8s audit logs + app logs), AU-6 (SIEM correlation +
  alertas), AU-9 (Immutable storage + encryption)
- **CA (Assessment):** CA-2 (pentests trimestrales + audits), CA-7 (Falco +
  Prometheus + SIEM)
- **CM (Configuration):** CM-2 (IaC con Terraform + GitOps), CM-3 (pull requests
  + approvals), CM-7 (distroless images + remove shells)
- **CP (Contingency):** CP-9 (Velero + pg_basebackup + Elasticsearch snapshots S3)
- **IA (Authentication):** IA-2 (MFA + OAuth2 + JWT), IA-5 (password policies +
  key rotation)
- **IR (Incident Response):** IR-4 (runbooks + PagerDuty 24/7), IR-6 (templates
  + escalación)
- **RA (Risk Assessment):** RA-5 (Trivy + Clair + Snyk)
- **SC (Communications):** SC-7 (WAF + Network Policies + Firewall), SC-8 (TLS
  1.3 + mTLS), SC-12 (Vault + cert-manager + rotation), SC-13 (AES-256 +
  RSA-2048+ + SHA-256+), SC-28 (LUKS + encrypted EBS volumes)
- **SI (Integrity):** SI-2 (patching SLA critical <7 días), SI-3 (image scanning
  + admission control), SI-4 (Falco + metrics + logs)

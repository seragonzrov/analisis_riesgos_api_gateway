con experiencia específica en arquitecturas cloud-native y Kubernetes.

El sistema bajo análisis es:
- API Gateway expuesto a internet (Kong / AWS API Gateway)

## PARTE 2: CONSTRUCCIÓN DEL KILL CHAIN COMPLETO

Construí un kill chain realista que encadene las amenazas más críticas en una secuencia de ataque coherente.

El kill chain debe:
- Comenzar con un vector de acceso inicial realista (CVE en Gateway, credencial comprometida, etc.)
- Progresar lógicamente de una fase a la siguiente
- Culminar en un objetivo de alto impacto (exfiltración de datos, disruption del servicio)
- Tener entre 6 y 10 pasos

Formato del kill chain:

```
KILL CHAIN: [Nombre del escenario de ataque]
Objetivo final: [Qué obtiene el atacante]
Tiempo estimado: [Horas/días para completar]

FASE 1 — [Tactic]: [Technique T-XXXX]
  → Acción: [Qué hace el atacante concretamente]
  → Prerequisito: [Qué necesita para este paso]
  → Artefacto: [Herramienta o payload utilizado]

FASE 2 — [Tactic]: [Technique T-XXXX]
  [...]

PUNTO DE DETECCIÓN: Entre FASE X y FASE Y existe la oportunidad de detección más temprana.
Control recomendado: [Qué alerta o control puede cortar la cadena acá]
```

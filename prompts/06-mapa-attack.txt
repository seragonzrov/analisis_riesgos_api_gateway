# Prompt 06: Mapeo MITRE ATT&CK

**Fecha:** 24/05/2026
**Herramienta:** Claude Sonnet 4.6 (Anthropic)
**Propósito:** Mapear las amenazas críticas al framework MITRE ATT&CK y construir el kill chain

---

## Prompt Utilizado

```
Eres un analista de threat intelligence especializado en MITRE ATT&CK
con experiencia en arquitecturas cloud-native y Kubernetes.
Contexto del sistema en 00-memory-bank.md.

Tenés las amenazas priorizadas en docs/05-priorizacion-dread.md.
Las 4 amenazas críticas (DREAD ≥ 40) son:
- TH18: API Flooding/DDoS (46/50)
- TH25: IDOR (42/50)
- TH06: SQL Injection (41/50)
- TH14: PII Exposure in Logs (40/50)

PARTE 1 — TABLA DE TÉCNICAS ATT&CK:
Generá una tabla con todas las técnicas ATT&CK relevantes para este sistema.
Columnas: ID Técnica | Nombre | Tactic | Mitigación
Usá técnicas de ATT&CK for Containers cuando aplique
(T1609, T1610, T1611, T1613, etc.)

PARTE 2 — KILL CHAIN COMPLETO:
Construí un kill chain realista que encadene las amenazas más críticas
desde acceso inicial hasta impacto final. Debe incluir Defense Evasion,
Lateral Movement e Impact además de las fases básicas.

Para cada fase:
- Número de fase y tactic ATT&CK
- Técnica con código T-XXXX
- Acción concreta del atacante en este sistema
- Vector o herramienta utilizada

Indicá el tiempo total estimado y los puntos críticos de detección.

PARTE 3 — HEAT MAP DE TACTICS:
Tabla con: Tactic | Técnicas ATT&CK | Cobertura de detección actual
Concluí con qué tactics tienen mayor brecha de detección.
```

---

## Response Summary

La IA generó:
- Tabla de **29 técnicas ATT&CK** distribuidas en 11 tactics:
  Initial Access (3), Execution (2), Persistence (3), Privilege Escalation (3),
  Defense Evasion (3), Credential Access (3), Discovery (3), Lateral Movement (2),
  Collection (2), Exfiltration (2), Impact (3)
- **Kill chain completo de 11 fases:**
  1. Initial Access (T1190) — CVE en Kong API Gateway
  2. Execution (T1609) — kubectl exec en container de Kong
  3. Persistence (T1525) — imagen Docker maliciosa en registry con backdoor
  4. Privilege Escalation (T1611) — container escape vía CVE-2019-5736 (runC)
  5. Defense Evasion (T1070.004) — eliminación de audit logs en nodo host
  6. Credential Access (T1552.007) — extracción K8s Secrets vía API
  7. Discovery (T1613) — enumeración de pods y servicios del cluster
  8. Lateral Movement (T1021) — acceso a PostgreSQL con credenciales extraídas
  9. Collection (T1005) — pg_dump completo de base de datos
  10. Exfiltration (T1537) — transferencia a S3 del atacante
  11. Impact (T1485) — eliminación de tablas para cubrir rastros
- Tiempo total estimado del ataque: ~3-6 horas sin detección
- 4 puntos críticos de detección: Fase 1 (WAF), Fase 4 (Falco),
  Fase 6 (K8s audit logs), Fase 9 (database activity monitoring)
- Heat map con Initial Access y Credential Access como tactics de mayor brecha

## Prompt Utilizado

```
Necesito crear el documento de información general para un análisis de riesgos
de seguridad formal. El sistema bajo análisis es una arquitectura de microservicios
con API Gateway.

Datos del equipo:
- Juan Lamolle, Serafín González, Fernando Rodríguez
- Fecha de inicio: 24/05/2026
- Fecha de entrega: 26/05/2026

Tecnologías del sistema:
- API Gateway: Kong / AWS API Gateway con OAuth2/JWT
- WAF: Web Application Firewall (ModSecurity/AWS WAF)
- Service Mesh: Istio con mTLS
- Microservicios: Users Service, Inventory Service, Orders Service
- Capa de datos: PostgreSQL, RabbitMQ, Elasticsearch
- Cache distribuido: Redis
- Orquestación: Kubernetes cluster
- Observabilidad: ELK Stack (Elasticsearch, Logstash, Kibana)

Generá un documento en Markdown con las siguientes secciones:

1. DESCRIPCIÓN DEL SISTEMA:
   - Descripción general de la arquitectura
   - Lista de componentes principales con su función

2. ALCANCE DEL ANÁLISIS:
   - Componentes INCLUIDOS en el análisis (con justificación)
   - Componentes FUERA DE ALCANCE en tabla con columnas:
     Componente | Razón de Exclusión

3. SUPUESTOS:
   Lista de supuestos de seguridad que se dan por verdaderos
   y que quedan fuera del análisis. Incluí supuestos sobre:
   - Infraestructura del cloud provider
   - Accesos del equipo de operaciones y desarrollo
   - Prácticas de desarrollo del código
   - Gestión de certificados TLS
   - Disaster recovery existente
   - Encriptación de tráfico entre zonas
   - Network policies del cloud provider

Formato Markdown con navegación al final
(enlace a doc anterior README.md y siguiente 02-inventario-activos.md).
```

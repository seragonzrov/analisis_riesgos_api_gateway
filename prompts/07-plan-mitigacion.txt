**Herramienta:** Claude Sonnet 4.6 (Anthropic)
**Propósito:** Diseñar controles de seguridad priorizados para mitigar las amenazas identificadas

## Prompt Utilizado

```
Eres un arquitecto de seguridad con experiencia en sistemas cloud-native,
Kubernetes y API security. Contexto del sistema en 00-memory-bank.md.

Tenés el análisis completo:
- 27 amenazas STRIDE en docs/04-analisis-stride.md
- Scores DREAD en docs/05-priorizacion-dread.md
- Kill chain en docs/06-mapa-attack.md

Amenazas críticas (DREAD ≥ 40):
- TH18: API Flooding/DDoS — 46/50
- TH25: IDOR — 42/50
- TH06: SQL Injection — 41/50
- TH14: PII Exposure in Logs — 40/50

Kill chain identificado (8 fases, ver docs/06-mapa-attack.md):
Initial Access → Execution → Persistence → Privilege Escalation →
Credential Access → Discovery → Collection → Exfiltration

Diseñá un plan de mitigación completo ejecutable en sprints de 2 semanas.

PARTE 1 — CONTROLES DE SEGURIDAD:
Para cada amenaza crítica o grupo relacionado, diseñá un control con:

CONTROL [ID] — [Nombre]
Prioridad: P0 / P1 / P2
Amenazas que mitiga: [TH-XX, ...]
Reducción de riesgo esperada: X%

DESCRIPCIÓN TÉCNICA: [qué hace y por qué es efectivo]

IMPLEMENTACIÓN PASO A PASO:
1. [paso concreto con tecnología específica]
2. ...

CONFIGURACIÓN DE REFERENCIA:
[YAML de K8s / regla Falco / comando / snippet]

CRITERIO DE ACEPTACIÓN:
[ ] [test que confirma que funciona]

ESFUERZO ESTIMADO: Xh | Sprint: 1/2/3
COSTO: [open source / licencia / costo cloud]

PARTE 2 — ROADMAP:
Sprint 1 (semanas 1-2): Controles P0
Sprint 2 (semanas 3-4): Controles P1
Sprint 3 (semanas 5-6): Controles P2 + pentest de validación
Para cada sprint: tabla con Control | Responsable | Dependencias | Definition of Done

PARTE 3 — ALINEACIÓN A FRAMEWORKS:
Tabla: Control | NIST CSF | ISO 27001 | OWASP API Top 10 | CWE mitigado

PARTE 4 — MÉTRICAS DE ÉXITO:
Por cada área de control: KPI | Umbral de alerta | Herramienta | Frecuencia

Preferir herramientas open source ya presentes en el stack.
Cada control P0 debe implementarse en menos de 16 horas de ingeniería.
```
